QA Analysis – Log Investigation & Database Queries

This repository contains the Quality Assurance tasks completed during Sprint 6 of a QA training program. The project focuses on log analysis using the command line and database validation using SQL queries.
The main objective was to investigate system behavior through log files, identify error responses, and analyze data stored in a relational database to validate business logic and system activity.

Project Objectives

Analyze server logs using Linux command-line tools
Extract and filter relevant log data
Identify HTTP error responses (400 and 500)
Organize filtered log data into structured files
Perform SQL queries to analyze and validate database information
Interpret data to support QA investigation and debugging
Log Analysis (Linux Console)
The first part of the project focuses on working with server logs using Linux commands.

Tasks included:

Searching logs using grep
Filtering logs by IP address
Extracting requests with specific HTTP status codes
Organizing logs into directories and files

Analyzing log results using tools like:

grep
head
tail
wc
mkdir

Example activities:

Finding requests from specific IP ranges
Extracting logs with 400 and 500 errors
Creating structured directories to store filtered logs
Reviewing samples of log data to investigate system issues

Database Analysis (SQL)

The second part of the project focuses on database queries to analyze taxi service data.

Key tasks included:

Counting the total number of vehicles in the database
Listing companies with fewer than 100 vehicles
Analyzing weather conditions from weather records
Calculating the number of trips completed by each taxi company within a specific time period

Example SQL operations used:

COUNT()
GROUP BY
HAVING
JOIN
ORDER BY
CASE statements for conditional data classification
